This project asks a simple question that a lot of landing pages avoid: which interface patterns are genuinely useful, and which ones are just business theatre with rounded corners?

Core question

Many landing pages are overloaded, manipulative, and cognitively exhausting while pretending to be polished. They do not clarify the offer. They perform confidence.

Design stance

The project argues for reduction instead of decoration. A landing page should behave more like a tool than a billboard. If an element does not help the user decide, understand, or move, it probably does not deserve the space.

Current notes

  • Strip away fake social proof and ornamental filler
  • Keep the hierarchy obvious even when the page is dense
  • Build with a visual system that signals function instead of mood-board drift
