This repository now behaves like a quiet publishing system instead of a framework demo. The site is generated from JSON content and HTML fragments, then shipped as plain static files to GitHub Pages.

Why strip it back

The previous Astro setup worked, but it was more machinery than the current site needed. For a site with a small route set and mostly editorial content, plain HTML and CSS are easier to edit, easier to inspect, and much harder to overcomplicate by accident.

What the rewrite keeps

  • A dedicated projects section
  • A dedicated writing section
  • Shared navigation and page layout
  • A watch-mode workflow for faster editing
  • GitHub Pages deployment through Actions

What it drops

  • Framework routing for a tiny site
  • Markdown collection plumbing
  • Astro-specific config and generated types
  • Any ambiguity about what the final HTML looks like before deploy
